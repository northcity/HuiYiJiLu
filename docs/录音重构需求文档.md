> 版本: v2.0 | 日期: 2026-04-01  
> 作者: 北城 | 状态: 待评审

## 一、重构目标

### 核心理念

**先做好录音 App，再做 AI 增值。**

当前项目录音结束后立刻触发「本地转写 → AI 总结 → 百炼工作流」的全套流水线，流程复杂、成本高、失败率不可控。本次重构将录音与 AI 处理彻底解耦：

```
旧流程: 录音结束 → 自动转写(本地) → 自动AI总结 → 自动百炼工作流 → 完成
新流程: 录音结束 → 直接保存 → 完成 ✅
                              ↓ (用户手动或自动触发)
                         AI 语音转录(阿里云API) → 完成 ✅
                              ↓ (用户按需选择)
                         AI 大模型处理(摘要/润色/待办/标题等) → 完成 ✅
```

### 三个阶段

| 阶段 | 目标 | 核心词 |
|------|------|--------|
| **阶段一：录音** | 可靠、安全、流畅、简洁的录音体验 | 录音 App |
| **阶段二：AI 转录** | 阿里云语音转文字 API（最经济方案） | 语音→文字 |
| **阶段三：AI 拓展** | 大模型按需处理（摘要/标题/润色/待办等） | AI 增值 |

### 3.2 状态机重新设计

```swift
enum MeetingStatus: String, Codable {
    // 阶段一：录音
    case recording       // 录音中
    case saved           // ✅ 新增：录音已保存（终态之一）
    
    // 阶段二：AI 转录
    case transcribing    // AI 转录中
    case transcribed     // ✅ 新增：转录完成（终态之一）
    
    // 阶段三：AI 处理
    case processing      // ✅ 新增：AI 大模型处理中（摘要/润色等）
    case completed       // 全部处理完成（终态）
    
    // 异常
    case failed          // 某步骤失败
}
```

**状态流转图：**

```
录音 → saved ──(用户触发转录)──→ transcribing → transcribed ──(用户触发AI)──→ processing → completed
                                      ↓ 失败                                     ↓ 失败
                                    failed                                      failed
```

### 4.2 录音前状态

- 显示当前日期
- 录音模式选择器（麦克风 / 内录 / 导入）保留现有逻辑
- 语言选择器前置到录音页（从设置页移出）
- 大按钮开始录音

### 4.3 录音中状态

**顶部信息区：**
- 左上角：红点 + `REC` 文字闪烁动画
- 日期显示（如「4月1日」，跟随系统语言）
- 实时音量条（现有波形优化为竖条风格，更紧凑）
- 语言选择器（录音中支持切换，用于后续转录的语言参数）
- 预留按钮位（文字转录 A、说话人识别 👤，灰色不可用，tooltip 提示"即将支持"）

**中央区域：**
- 圆形波纹动画（根据音量大小，圆环呼吸式缩放 + 多层淡色圆环）
- 中心大字计时器（保持现有 `00:20` 风格）

**控制按钮区：**
- 左：取消/关闭(✕) — 改为圆形图标按钮，与竞品一致
- 中：停止录音(■) — 保持红色圆形
- 右：暂停/继续(⏸/▶) — 保持圆形灰色

**底部工具栏（新增）：**
- 🚩 录音打点 — 一键添加时间标记，长按可输入标签
- 📝 添加笔记 — 弹出文字输入框，关联当前录音时间点
- 📷 拍照/相册 — 调用相机或相册选取照片，关联当前录音时间点

### 4.4 录音结束

**核心变更：录音结束后直接保存，不再自动触发任何 AI 处理。**

```swift
// 旧逻辑（删除）
func stopAndSave() {
    // 1. 停止录音 → 2. 创建Meeting(status=.transcribing) → 3. MeetingProcessingService.processInBackground() → 4. dismiss
}

// 新逻辑
func stopAndSave() {
    // 1. 停止录音
    // 2. 创建 Meeting(status=.saved)，保存音频文件名、时长、语言、打点、笔记、照片
    // 3. 显示保存成功动画（✅ 录音已保存，无 AI 处理提示）
    // 4. dismiss 返回首页
}
```

### 4.5 录音模式保留

| 模式 | 说明 | 状态 |
|------|------|------|
| 麦克风 | AVAudioRecorder，现有逻辑 | ✅ 保留 |
| 系统内录 | ReplayKit Broadcast Extension | ✅ 保留 |
| 导入录音 | 从文件 App 导入音频 | ✅ 保留（现有 MeetingListView 的导入功能） |

## 五、AI 语音转录（阶段二）

### 5.1 转录时机

**不再自动触发。** 用户在会议详情页手动点击「转录」按钮，或在设置中开启「录音结束自动转录」。

### 5.2 转录服务切换

| 方案 | 说明 | 成本 | 优先级 |
|------|------|------|--------|
| **阿里云语音转文字 API** | Paraformer / SenseVoice 等最经济模型 | 低 | ✅ 默认方案 |
| 本地 SFSpeechRecognizer | 现有逻辑 | 免费但质量有限 | 保留为备选 |
| 通义听悟 TingWu | 功能最全但成本高 | 高 | 保留为高级功能 |

### 5.3 阿里云语音转文字 API 接入要点

> 用户稍后提供具体模型和文档，以下为架构预留：

```swift
// 新建 Services/AliyunASRService.swift
class AliyunASRService: ObservableObject {
    /// 调用阿里云语音转文字 API
    /// - Parameters:
    ///   - audioFileURL: 本地音频文件（或先上传 OSS 拿到 URL）
    ///   - language: 语言代码（zh / en / auto）
    ///   - enableSpeakerDiarization: 是否启用说话人分离
    ///   - enableMultiLanguage: 是否启用多语言混合识别
    /// - Returns: 转录结果（含时间戳、说话人标签等）
    func transcribe(
        audioFileURL: URL,
        language: String,
        enableSpeakerDiarization: Bool,
        enableMultiLanguage: Bool
    ) async throws -> ASRResult
}

struct ASRResult {
    let text: String                          // 纯文本
    let segments: [ASRSegment]                // 带时间戳的分段
    let speakers: [String]                    // 识别到的说话人列表
}

struct ASRSegment: Codable {
    let startTime: TimeInterval
    let endTime: TimeInterval
    let text: String
    let speaker: String?                      // 说话人标签
    let language: String?                     // 识别到此段的语言
    let confidence: Double?
}
```

### 5.4 转录时需提前支持的能力

这些能力需要在 ASR API 调用时通过参数启用，而非后期 AI 大模型处理：

| 能力 | 说明 | 实现方式 |
|------|------|---------|
| **多语言识别** | 中英混合、中日混合等 | ASR API 参数 `language=auto` 或 `language=zh-en` |
| **说话人分离** | 区分不同说话人 | ASR API 参数 `speaker_diarization=true` |
| **时间戳** | 每句话的起止时间 | ASR API 默认返回 |
| **标点恢复** | 自动添加标点 | ASR API 参数 |

## 六、AI 大模型处理（阶段三）

### 6.1 触发方式

转录完成后，用户在会议详情页按需选择 AI 处理功能。可一键全部执行，也可单独选择。

### 6.2 AI 功能清单

| 功能 | 说明 | 调用方式 | 优先级 |
|------|------|---------|--------|
| **自动标题** | 根据转录内容生成简洁标题 | AI 大模型 (Qwen) | P0 |
| **章节总结** | 按主题/时间段分章节，每章一句话总结 | AI 大模型 | P0 |
| **全文摘要** | 2-3 句话概括会议核心 | AI 大模型 | P0 |
| **待办提取** | 提取行动项 + 责任人 + 优先级 | AI 大模型 | P0 |
| **关键决策** | 提取会议中做出的决策 | AI 大模型 | P1 |
| **文字润色** | 将口语化转录润色为书面语 | AI 大模型 | P1 |
| **翻译** | 将转录/摘要翻译为其他语言 | AI 大模型 | P2 |
| **自定义提问** | 用户输入自定义 prompt 处理转录内容 | AI 大模型 | P2 |

### 6.3 AI 服务保留与降级

| 服务 | 当前角色 | 重构后角色 |
|------|---------|-----------|
| **AIService** | 自动生成摘要/标题/行动项 | ✅ 保留，改为按需调用 |
| **BailianWorkflowService** | 自动生成图文纪要 HTML | ⏸ 代码保留，UI 入口隐藏，作为高级功能备用 |
| **TingWuService** | 通义听悟智能纪要 | ⏸ 代码保留，UI 入口隐藏，作为高级功能备用 |
| **TranscriptionService** | 本地 SFSpeechRecognizer | ✅ 保留为免费备选方案 |
| **AliyunASRService（新建）** | — | ✅ 新建，默认转录方案 |

## 九、MeetingProcessingService 重构

### 9.1 现有逻辑（需重构）

```swift
// 现有：一条流水线自动全跑
func processInBackground(meeting:, modelContext:) {
    1. transcribe (SFSpeechRecognizer)    ← 删除自动执行
    2. generateSummary (AIService)         ← 删除自动执行
    3. generateRichNotes (百炼 or AI)       ← 删除自动执行
}
```

### 9.2 新逻辑：按需分步调用

```swift
// 重构为独立的按需方法
class MeetingProcessingService {

    /// 阶段二：AI 转录（用户手动触发或自动触发）
    func transcribe(meeting: Meeting, modelContext: ModelContext) async {
        meeting.status = .transcribing
        // 根据 asr_provider 设置选择：
        //   - "aliyun" → AliyunASRService.transcribe()
        //   - "local"  → TranscriptionService.transcribe()
        // 转录完成后:
        meeting.transcript = result.text
        meeting.isTranscribed = true
        meeting.status = .transcribed
    }

    /// 阶段三：AI 大模型处理（用户按需选择功能）
    func processWithAI(
        meeting: Meeting, 
        modelContext: ModelContext,
        tasks: Set<AITask>   // [.title, .summary, .chapters, .actionItems, .polish]
    ) async {
        meeting.status = .processing
        // 逐个执行选中的 AI 任务
        // 结果写入 meeting 对应字段
        meeting.isAIProcessed = true
        meeting.status = .completed
    }
    
    enum AITask {
        case title, summary, chapters, actionItems, decisions, polish, translate
    }
}
```

## 十二、技术风险与注意事项

### ⚠️ SwiftData 迁移

新增字段需要确保 SwiftData 轻量迁移兼容。所有新字段必须有默认值：
```swift
var languageCode: String = "zh-Hans"
var bookmarks: String = "[]"
var notes: String = "[]"
var photoFileNames: String = "[]"
var sourceType: String = "microphone"
var transcriptProvider: String = ""
var isTranscribed: Bool = false
var isAIProcessed: Bool = false
```

### ⚠️ 照片存储

录音中拍摄的照片保存到 `Documents/MeetingPhotos/meeting_{id}/` 目录，文件名记录在 Meeting.photoFileNames JSON 中。需处理存储空间和清理逻辑。

### ⚠️ 阿里云 ASR API Key 复用

阿里云语音转文字 API 和大模型 API 可能使用同一个 DashScope API Key。设置页应允许复用同一 Key 或分别配置。

### ⚠️ 录音可靠性

录音是 App 的核心功能，重构过程中不能影响现有录音流程的稳定性：
- AudioRecorderService 不做改动
- SystemAudioRecorderService 不做改动  
- 新功能（打点/笔记/拍照）为附加功能，不影响录音主流程

### ⚠️ 向后兼容

已有的 Meeting 数据（status = completed）需兼容。旧数据的 `isTranscribed` 和 `isAIProcessed` 应根据 transcript/summary 是否非空自动推断。

## 十三、总结

本次重构的核心思路：

1. **录音优先** — 先做最可靠的录音 App，打点/笔记/拍照增强录音体验
2. **AI 后置** — 录音结束即保存完成，AI 转录和处理按需触发
3. **成本优化** — 百炼工作流暂时隐藏，使用阿里云最经济的 ASR API
4. **能力预留** — 说话人分离、多语言识别在 ASR 层支持，为后续功能铺路
5. **代码保留** — 百炼/听悟服务代码不删除，随时可恢复为高级功能
