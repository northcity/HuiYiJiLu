# 云雀记 App 重构实施方案

## 一、重构概览

### 当前问题
1. 会议详情页5个Tab（Summary/Transcript/Tasks/图文纪要/智能纪要）内容高度重叠
2. 3路AI输出（AIService/BailianWorkflow/TingWu）各自独立，用户不知道看哪个
3. 录音结束后用户必须等待处理完成才能离开RecordingView
4. 没有底部TabBar，待办散落在各会议内部，无法跨会议管理
5. 设置页暴露大量API Key配置，普通用户困惑

### 改动范围

| 优先级 | 事项 | 改动文件 |
|--------|------|---------|
| P0 | 会议详情页重写（5Tab→长滚动页+合并纪要） | `MeetingDetailView.swift` |
| P0 | 录音后处理后台化 | `RecordingView.swift`, `MeetingListView.swift` |
| P1 | 新增底部TabBar（会议/待办/我的） | `ContentView.swift`, 新建 `TaskListView.swift`, `ProfileView.swift` |
| P1 | 首页新增每日洞察卡片 | `MeetingListView.swift`, `AIService.swift` |
| P1 | 设置页简化 | `SettingsView.swift` |

## 二、P0-1 + P0-2：会议详情页重构

### 改动前
```
Picker 5-Tab: Summary | Transcript | Tasks | 图文纪要 | 智能纪要
每个Tab独立展示，内容重叠，用户切换成本高
```

### 改动后
```
ScrollView 长滚动页：
  ├── 会议头部信息（日期、时长）
  ├── 音频播放器
  ├── 处理中状态（条件显示）
  ├── AI纪要卡片（合并3路输出为最佳1份）
  │    ├── 优先级：TingWu智能纪要 > 百炼图文纪要 > AI Summary
  │    └── 支持展开/折叠
  ├── 关键要点（如有）
  ├── 待办清单
  ├── 原始转写（默认折叠，点击展开）
  └── 底部操作区（重新生成、分享、查看TingWu详情）
```

### 合并逻辑
- 如果有 `tingwuNotes`（通义听悟），优先展示，并提供"查看详情"入口进入 TingWuDetailView
- 否则如果有 `richNotes`（百炼/AI生成），展示 HTML 或 Markdown
- 否则展示 `summary` + `keyPoints`
- 三者都没有时显示空状态+生成按钮

## 三、P0-3：录音后处理后台化

### 改动前
```
RecordingView: stopAndProcess() → 显示 processingView → 等待完成 → dismiss()
用户必须盯着"转写中...""AI分析中..."等完成才能离开
```

### 改动后
```
RecordingView: stopRecording() → 创建Meeting(status=.transcribing) → 立即dismiss()
后台: Meeting创建后自动触发处理流水线
MeetingListView: 卡片上显示实时处理状态（转写中/分析中/已完成）
```

### 实现方式
- 在 RecordingView 中：停止录音后创建 Meeting、保存到 SwiftData、立即 dismiss
- 新增 `MeetingProcessingService`：接收 Meeting，后台执行转写→总结→纪要
- MeetingListView 已有状态显示逻辑（statusPill），无需改动

## 四、P1-1：底部TabBar

### 新增结构
```
ContentView (TabView)
├── Tab 1: 会议 → MeetingListView（现有）
├── Tab 2: 待办 → TaskListView（新建）
│    ├── 全部待办（跨会议聚合）
│    ├── 按状态筛选（未完成/已完成）
│    └── 点击待办可跳转到所属会议
└── Tab 3: 我的 → ProfileView（新建）
     ├── 使用统计（本月会议数/总时长/待办完成率）
     ├── 设置入口（合并现有SettingsView）
     └── 关于/版本
```

## 五、P1-2：首页每日洞察卡片

### 位置
MeetingListView 首页顶部，headerView 下方

### 内容
```
┌─────────────────────────────────┐
│ 今日洞察                         │
│ 今天2个会议，3个待办未完成。       │
│ 产品评审会的方案需要你今天确认。    │
│                    [查看待办 →]   │
└─────────────────────────────────┘
```

### 实现
- 在 AIService 中新增 `generateDailyInsight()` 方法
- 根据当天会议的 summary 和未完成 actionItems 生成一句话洞察
- 无AI Key时使用本地统计文案兜底

## 六、P1-3：设置页简化

### 改动前
设置页直接展示：AI提供商 + 模型 + 输出风格 + 录音模式 + 云服务配置（DashScope/TingWu/百炼/OSS 四个子页面）

### 改动后
```
设置页（普通模式）：
├── AI 模型选择
├── 输出风格
├── 录音模式 & 语言
├── 导出格式
├── 数据管理
└── 关于（版本号，连续点击5次解锁开发者模式）

设置页（开发者模式）：
├── 以上所有 +
├── 云服务配置（DashScope/TingWu/百炼/OSS）
└── API Base URL
```
